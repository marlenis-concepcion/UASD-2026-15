# SKILL: CREAR PROCEDIMIENTO
## `/crear-procedimiento`

**Propósito:** Guía paso a paso para crear un nuevo procedimiento desde cero  
**Tiempo estimado:** 3-4 semanas  
**Complejidad:** Media  

---

## ¿QUÉ HACE ESTE SKILL?

Te guía para crear un procedimiento completo con los 10 componentes obligatorios, validado legalmente y optimizado para sector público.

---

## INICIO RÁPIDO

```bash
/crear-procedimiento [nombre-del-procedimiento]

Ejemplo:
/crear-procedimiento solicitud-permisos-especiales
```

---

## PASOS

### PASO 1: DIAGNÓSTICO (Día 1-2)

**Responsable:** Yisairis + Responsable del área

**Tareas:**
- [ ] Entrevista rápida con operativos (30 min)
- [ ] Revisar documentación existente
- [ ] Identificar 3 personas clave
- [ ] Problemas identificados: _________________

**Salida:** Documento de diagnóstico (1 página)

---

### PASO 2: ESTRUCTURA BASE (Día 3-4)

**Responsable:** Yisairis

**Tareas:**
- [ ] Copiar PLANTILLA_PROCEDIMIENTO_ESTANDAR.md
- [ ] Llenar secciones 1-6 (Objetivos, Alcance, Base Legal, etc)
- [ ] Investigar leyes aplicables
- [ ] Identificar responsabilidades

**Salida:** Procedimiento con secciones 1-6 completas

---

### PASO 3: ACTIVIDADES DETALLADAS (Día 5-7)

**Responsable:** Yisairis + Operativos

**Tareas:**
- [ ] Describir paso a paso (Sección 7)
- [ ] Crear diagrama de flujo (Sección 8)
- [ ] Definir registros (Sección 9)
- [ ] Preparar anexos (Sección 10)

**Herramientas:**
- Google Docs para redacción
- Draw.io para diagramas
- Google Sheets para matrices

**Salida:** Procedimiento 100% redactado

---

### PASO 4: VALIDACIÓN LEGAL (Día 8)

**Responsable:** Asesor Legal

**Tareas:**
- [ ] Revisar base legal
- [ ] Verificar conformidad
- [ ] Resolver dudas
- [ ] Aprobar por escrito

**Usar:** MATRIZ_LEGALIDAD.md

**Salida:** Aprobación legal escrita

---

### PASO 5: VALIDACIÓN OPERATIVA (Día 9-10)

**Responsable:** Responsables de área + Yisairis

**Tareas:**
- [ ] Reunión de revisión (2 horas)
- [ ] Feedback de usuarios finales
- [ ] Ajustes necesarios (máx 24h)
- [ ] Aprobación ejecutiva

**Salida:** Versión 1.0 del procedimiento

---

## CHECKLIST DE COMPLETITUD

**10 Componentes:**
- [ ] 1. Objetivos (general + 3-5 específicos)
- [ ] 2. Alcance (áreas, niveles, limitaciones)
- [ ] 3. Base Legal (leyes + artículos específicos)
- [ ] 4. Responsabilidades (matriz de roles)
- [ ] 5. Términos y Definiciones (glosario)
- [ ] 6. Políticas (3-5 políticas aplicables)
- [ ] 7. Descripción de Actividades (paso a paso)
- [ ] 8. Diagrama de Flujo (visual)
- [ ] 9. Registros (qué se documenta, cómo, dónde)
- [ ] 10. Anexos (formatos, ejemplos, capacitación)

**Calidad:**
- [ ] Lenguaje claro (nivel secundaria)
- [ ] Sin ambigüedades
- [ ] Tiempos realistas
- [ ] Costo estimado
- [ ] Responsables claros

**Legalidad:**
- [ ] Validado por asesor legal
- [ ] 100% conforme a normas
- [ ] Derechos laborales protegidos
- [ ] Privacidad de datos garantizada

**Eficiencia:**
- [ ] Reducción de tiempo (meta -20%)
- [ ] Reducción de costo (meta -15%)
- [ ] Recursos optimizados
- [ ] Factor humano considerado

## PLANTILLA RÁPIDA

```markdown
# PROCEDIMIENTO: [Nombre]

Versión: 1.0
Fecha: [Hoy]
Aprobado: [Nombre]

## 1. OBJETIVOS
**General:** [Objetivo amplio en presente]
**Específicos:**
1. [Objetivo medible 1]
2. [Objetivo medible 2]

## 2. ALCANCE
[Describir quién, dónde, cuándo aplica]

## 3. BASE LEGAL
[Tabla con leyes y artículos]

[Continuar con secciones 4-10...]
```

---

## ERRORES A EVITAR

❌ **No hacer esto:**
- Crear sin validar legalmente
- Procedimiento demasiado largo (+ de 15 páginas)
- Responsabilidades duplicadas
- Sin ejemplos o anexos
- Sin diagrama de flujo
